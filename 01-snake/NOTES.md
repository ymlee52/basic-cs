## 처음 본 C 문법

> 단계마다 새로 만난 문법을 한 줄씩.

- **Stage 0**
  - `#include <stdio.h>` — 전처리기 지시문. 표준 입출력 함수 쓰려면 필요.
  - `int main(void) { ... }` — 프로그램의 진입점. 정수를 반환, 인자 없음.
  - `printf("...\n")` — 화면 출력. `\n`은 줄바꿈.
  - `return 0;` — 정상 종료. 0이 아니면 에러 코드. 모든 문장은 `;`로 끝.

- **Stage 1**
  - `#define ROWS 6` — 매크로 상수. 전처리 단계에서 단순 치환되며, 관습적으로 대문자.
  - `char board[ROWS][COLS];` — 2D 배열 선언. 자료형 + 이름 + 차원 + `;`.
  - `for (int r = 0; r < ROWS; r++) { ... }` — for 루프 (초기화 / 조건 / 증감). 안에서 선언한 `int r`은 그 루프에서만 살아있음.
  - `if (조건) { ... } else { ... }` — 조건 분기. 조건은 `||`(OR), `&&`(AND)로 묶을 수 있음.
  - `'#'` (작은따옴표) — 문자 1개. `"#"` (큰따옴표) — 문자열. C에서 둘은 다른 타입.
  - `putchar(c)` — 문자 1개를 화면에 출력. `printf("%c", c)`와 동일.

## 깨달은 CS 개념

> 단계마다 "아 이거구나" 한 것을 짧게.

- **Stage 0**
  - `.c` → `.exe`는 한 번에 되는 게 아니라 **전처리 → 컴파일 → 어셈블 → 링크** 단계를 거친다. `gcc`가 이걸 한 줄로 묶어줄 뿐.
  - **프로세스의 수명 = 콘솔 창의 수명.** `main`이 `return`하면 프로세스가 끝나고, Windows가 자동으로 콘솔 창을 닫는다. 그래서 exe를 더블클릭하면 창이 깜빡이고 사라진다.
  - 터미널에서 `./파일명`으로 실행하면 **이미 열려있는 터미널 프로세스**가 출력을 받아주기 때문에 창이 안 닫힌다.

- **Stage 1**
  - **2D 배열도 메모리에선 1차원이다.** `char board[6][18]`은 `[0행 0열][0행 1열]...[0행 17열][1행 0열]...` 순으로 한 줄로 펼쳐 저장됨 (행 우선, row-major). `board[r][c]`의 주소 = `base + r * COLS + c`.
  - **데이터(메모리)와 표현(화면)은 분리되어 있다.** `board[r][c] = '#'`은 RAM에 글자를 넣을 뿐, 화면엔 아무것도 안 뜸. `putchar`/`printf`로 명시적으로 출력해야 비로소 stdout으로 보내짐. → 게임 루프에서 "상태 갱신"과 "렌더링"이 분리되는 패턴의 시작.
  - **매크로(`#define`)는 변수가 아니다.** 컴파일 시작 전에 전처리기가 단순 치환만 함. 그래서 메모리도 안 차지하고, 런타임에 바꿀 수도 없음.

- **Stage 0**: exe 파일을 더블클릭했더니 창이 떴다 사라짐. → 원인: 프로세스가 끝나면 창도 닫힘. → 해결: 터미널을 먼저 열고 거기서 `./01-snake/snake` 실행.
