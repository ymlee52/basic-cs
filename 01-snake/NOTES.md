# 01-snake — 학습 노트

터미널에서 화살표키로 조작하는 스네이크 게임을 만들면서 C 문법과 CS 개념을 익힌다.

## 진행 현황

| Stage | 내용 | CS 개념 | 상태 |
|---|---|---|---|
| 0 | Hello World 컴파일 | 컴파일 → 링크 → 실행 흐름 | ✅ |
| 1 | 정적 보드 그리기 | 2D 배열, 메모리 레이아웃 | ⬜ |
| 2 | 키 한 번 받아 이동 | 표준 입출력, 버퍼링, ANSI escape | ⬜ |
| 3 | 게임 루프 (자동 전진) | 무한 루프, 틱(tick), Sleep vs busy-wait | ⬜ |
| 4 | 사과 먹고 길어지기 | 구조체, malloc/free, 큐 | ⬜ |
| 5 | 충돌·게임오버·점수 | 상태 머신, 난수 | ⬜ |
| 6 (선택) | 배열 → 연결 리스트 리팩터 | 포인터, 연결 리스트 | ⬜ |

## 단계별 체크 질문

- **Stage 0**: `.c` → `.exe` 사이에 어떤 단계를 거치지?
- **Stage 1**: `board[3][5]`의 메모리 주소는 어떻게 계산되지?
- **Stage 2**: 왜 `getchar()`는 Enter를 눌러야 반응할까?
- **Stage 3**: `while(1)`에 `Sleep`이 없으면 CPU는 어떻게 되지?
- **Stage 4**: `malloc`한 메모리를 `free` 안 하면 무슨 일이?
- **Stage 5**: `srand`를 안 부르면 `rand()`가 매번 같은 이유는?
- **Stage 6**: 뱀 몸통은 배열 vs 연결 리스트 중 어느 쪽이 자연스럽지? 왜?

## 처음 본 C 문법

> 단계마다 새로 만난 문법을 한 줄씩.

- **Stage 0**
  - `#include <stdio.h>` — 전처리기 지시문. 표준 입출력 함수 쓰려면 필요.
  - `int main(void) { ... }` — 프로그램의 진입점. 정수를 반환, 인자 없음.
  - `printf("...\n")` — 화면 출력. `\n`은 줄바꿈.
  - `return 0;` — 정상 종료. 0이 아니면 에러 코드. 모든 문장은 `;`로 끝.

## 깨달은 CS 개념

> 단계마다 "아 이거구나" 한 것을 짧게.

- **Stage 0**
  - `.c` → `.exe`는 한 번에 되는 게 아니라 **전처리 → 컴파일 → 어셈블 → 링크** 단계를 거친다. `gcc`가 이걸 한 줄로 묶어줄 뿐.
  - **프로세스의 수명 = 콘솔 창의 수명.** `main`이 `return`하면 프로세스가 끝나고, Windows가 자동으로 콘솔 창을 닫는다. 그래서 exe를 더블클릭하면 창이 깜빡이고 사라진다.
  - 터미널에서 `./파일명`으로 실행하면 **이미 열려있는 터미널 프로세스**가 출력을 받아주기 때문에 창이 안 닫힌다.

## 삽질 기록

> 에러 메시지 / 오타 / 세그폴트 → 어떻게 해결했나.

- **Stage 0**: exe 파일을 더블클릭했더니 창이 떴다 사라짐. → 원인: 프로세스가 끝나면 창도 닫힘. → 해결: 터미널을 먼저 열고 거기서 `./01-snake/snake` 실행.
