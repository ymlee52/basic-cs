# Stage 4-(2) — 사과 랜덤 배치

> 보드 어딘가에 사과 `*`를 띄운다. 매 실행마다 다른 자리. 처음으로 **난수**를 만난다. 먹기 판정은 아직 없음.

## 오늘의 목표

- 보드 안쪽 어딘가에 사과 `*`를 한 개 표시.
- 매 실행마다 다른 자리에 뜨도록.
- 뱀이 사과 위를 지나가도 사과는 그대로 남음 (먹기 판정은 다음 단계).
- `q`로 종료.

기대 화면:
```
##################
#................#
#.....*..........#       ← 매 실행마다 자리가 바뀜
#......O.........#
#................#
#................#
#................#
##################
```

---

## 배경 이야기 — 의사난수란 무엇인가

컴퓨터는 사실 **진짜 난수**를 못 만든다. CPU는 결정적 기계라서 같은 입력이면 같은 출력만 나온다. 우리가 `rand()`라고 부르는 건 **의사난수**(pseudo-random) — 어떤 시작값(시드)에서 정해진 수식으로 다음 값을 계산해 가는 수열이다. 겉보기엔 무작위처럼 보이지만, 시드가 같으면 매번 똑같은 순서가 나온다.

```
srand(42); rand() → 8453, 4321, 99, 7102, ...
srand(42); rand() → 8453, 4321, 99, 7102, ...   ← 같은 시드 → 같은 수열
srand(99); rand() → 1287, 7654, 33, 5511, ...   ← 다른 시드 → 다른 수열
```

매 실행마다 다른 사과 자리를 원하면 시드를 매번 다르게 줘야 한다. 가장 흔한 방법: **현재 시각**(`time(NULL)`)을 시드로. 매 초마다 값이 달라지니까 실행 시각이 다르면 시드도 다르다.

### 시드를 안 주면? — `srand` 호출 생략의 결과

`srand`를 한 번도 안 부르면 시드는 기본값 `1`로 시작한다. `rand()`만 부르면 **매 실행 같은 수열**이 나온다. 다른 사과 자리를 원하면 반드시 `srand(time(NULL))`을 `main` 시작 직후 한 번.

### 재현성 vs 변동성

같은 시드 = 같은 결과는 **버그가 아니라 기능**이다.
- **재현성이 중요한 곳**(테스트, 시뮬레이션, 절차적 생성 맵): **고정 시드**로 같은 결과 보장. "시드 42로 만든 던전"을 다른 사람도 똑같이 볼 수 있음.
- **변동성이 중요한 곳**(게임 플레이, 뽑기): 매번 다른 시드로 사용자 경험 다양화.

게임의 사과 자리는 변동성 쪽 — `time(NULL)`을 시드로.

### 모듈로 — 0 ~ N-1 범위로 좁히기

`rand()`는 0 ~ `RAND_MAX`(보통 32767) 사이 정수를 준다. 너무 큼. 보드 좌표 범위(테두리 안: 1 ~ ROWS-2)로 좁히려면 **모듈로 연산자 `%`**.

```
rand() % N → 0 ~ N-1 사이 정수
1 + rand() % (ROWS - 2) → 1 ~ ROWS-2 사이 (테두리 안)
```

> 사실 `rand() % N`은 N이 `RAND_MAX + 1`의 약수가 아니면 약하게 편향된다. 실무 코드라면 더 정교한 방법을 쓰지만 학습 단계에선 무시.

### 사과를 매 프레임 다시 박는 이유

사과는 안 움직이지만 한 번만 `board`에 박아두면 **뱀이 그 자리를 지나갈 때 사라진다** — 직전 꼬리 자리를 `.`으로 되돌리는 차분 렌더링이 사과도 같이 지워버리기 때문. 그래서 매 프레임 사과 자리에 `*`를 다시 박아야 한다.

박는 순서도 중요: **사과 → 머리** 순서로. 머리가 사과 위에 있는 프레임에 사과가 머리를 덮어버리면 안 됨. 머리가 위에 보이게.

---

## 새 CS 개념

- **의사난수(PRNG)**: 결정적 알고리즘이 만드는 "무작위처럼 보이는" 수열. 시드로 결정.
- **시드(seed)**: 수열의 시작점. 같으면 같은 결과. 매번 다른 결과를 원하면 시각·외부 입력 등 변동값을 시드로.
- **재현성 vs 변동성**: 둘 다 의도된 설계 선택. 시드를 어떻게 정하느냐로 결정.
- **모듈로 편향**: `rand() % N`의 분포가 완전 균등이 아닐 수 있다는 사실. 학습 단계에선 무시.

---

## 새 C 문법

### `srand(unsigned int seed);`
- `<stdlib.h>`. 난수 수열의 시작점 설정. 프로그램당 한 번만 부르는 게 보통.
- **루프 안에서 부르지 말 것** — 시드를 매번 바꾸면 `rand()`가 거의 같은 값만 반환(시각이 거의 안 바뀌니까).

### `int rand();`
- `<stdlib.h>`. 다음 의사난수 반환. 범위는 `0` ~ `RAND_MAX`(보통 32767, 환경마다 다를 수 있음).

### `time_t time(time_t *t);`
- `<time.h>`. 현재 시각을 **Unix epoch 초**(1970-01-01 00:00:00 UTC부터 흐른 초)로 반환.
- 인자에 `NULL`을 주면 그냥 반환만 함. `srand(time(NULL))`처럼 한 줄 관용구.
- `time_t`는 정수형(보통 `long`). `srand`가 `unsigned int`를 받으므로 **암묵적 변환**이 일어남 — 학습 단계에선 무시 OK.

### `%` (모듈로)
- 나머지 연산자. `a % b` = a를 b로 나눈 나머지. **정수에만** 사용. 실수는 `fmod` (별도 함수).

### 구조체 초기화 `Cell apple = { r값, c값 };`
- 중괄호 안에 필드 순서대로 값. `apple.r = ...`로 한 줄씩 쓰는 것과 동일.
- C99부터는 필드명으로도 가능: `Cell apple = { .r = 3, .c = 5 };` (가독성↑, 순서 무관).

## 코드

`01-snake/snake.c`를 다음으로 덮어쓴다.

```c
#include <windows.h>
#include <stdio.h>
#include <conio.h>
#include <stdlib.h>
#include <time.h>

#define ROWS 8
#define COLS 18

typedef struct {
  int r;
  int c;
} Cell;

#define MAX_LEN (ROWS * COLS)

int main(void) {
  SetConsoleOutputCP(65001);

  // (추가) 의사난수 시드 — 매 실행마다 다른 시각이라 다른 사과 자리.
  // 반드시 main 맨 처음 한 번. 루프 안에서 부르면 시드가 매번 바뀌어 망가짐.
  srand(time(NULL));

  char board[ROWS][COLS];

  int dr = 0, dc = 1;

  for (int r = 0; r < ROWS; r++) {
    for (int c = 0; c < COLS; c++) {
      if (r == 0 || r == ROWS - 1 || c == 0 || c == COLS - 1) {
        board[r][c] = '#';
      } else {
        board[r][c] = '.';
      }
    }
  }

  Cell *body = malloc(sizeof(Cell) * MAX_LEN);
  int length = 1;

  body[0].r = ROWS / 2;
  body[0].c = COLS / 2;

  Cell prev_tail = body[0];

  // (추가) 사과 한 개 랜덤 배치 — 테두리 안(행·열 모두 1 ~ ROWS-2 / 1 ~ COLS-2).
  // rand() % N 은 0 ~ N-1, 거기에 +1 해서 1 ~ N-2 범위로 옮김 (N = ROWS-2 같은 식).
  // 머리(보드 한가운데)와 겹칠 가능성은 일단 무시 (약 1% 확률, 다음 단계에서 처리).
  Cell apple;
  apple.r = 1 + rand() % (ROWS - 2);
  apple.c = 1 + rand() % (COLS - 2);

  int running = 1;

  while (running) {
    printf("\033[H");

    // === 차분 렌더링 ===
    board[prev_tail.r][prev_tail.c] = '.';      // 직전 꼬리 지우기
    // (추가) 사과를 매 프레임 다시 박는다 — 차분 렌더링이 '.'으로 덮어쓴 자리도 복구.
    // 박는 순서가 핵심: 사과 → 머리. 머리가 사과 위에 있어도 'O'가 보이게.
    board[apple.r][apple.c] = '*';
    board[body[0].r][body[0].c] = 'O';

    for (int r = 0; r < ROWS; r++) {
      for (int c = 0; c < COLS; c++) {
        putchar(board[r][c]);
      }
      putchar('\n');
    }

    printf("이동: wasd, 종료: q\n");

    if (_kbhit()) {
      int key = _getch();
      switch (key) {
        case 'w': dr = -1, dc =  0; break;
        case 's': dr =  1, dc =  0; break;
        case 'a': dr =  0, dc = -1; break;
        case 'd': dr =  0, dc =  1; break;
        case 'q': running = 0; break;
      }
    }

    prev_tail = body[length - 1];

    for (int i = length - 1; i > 0; i--) {
      body[i] = body[i - 1];
    }

    body[0].r += dr;
    body[0].c += dc;

    if (body[0].r < 1)        body[0].r = 1;
    if (body[0].r > ROWS - 2) body[0].r = ROWS - 2;
    if (body[0].c < 1)        body[0].c = 1;
    if (body[0].c > COLS - 2) body[0].c = COLS - 2;

    Sleep(200);
  }

  free(body);
  return 0;
}
```

확인 사항:
- 사과 `*`가 매번 다른 자리에 뜨는가? (`q`로 종료 후 다시 실행 → 자리 바뀜)
- 뱀이 사과 위를 지나가도 사과가 그대로 유지되는가? (이번 단계 핵심 — 차분 렌더링 + 매 프레임 재박기)

## 흔히 빠지는 함정

- **`srand` 호출 안 함** → 매 실행 같은 자리. 시드 기본값 1 사용.
- **`srand`를 루프 안에서 호출** → 매 프레임 시드 재설정 → `rand()`가 거의 같은 값(시각이 거의 안 바뀜) → 사과가 한 자리에 갇힘.
- **`rand() % ROWS`로 범위 잡기** → 0행 또는 ROWS-1행에 사과가 박힐 수 있음(테두리 위). `1 + rand() % (ROWS - 2)`로 1~ROWS-2.
- **사과 박기를 머리 박기 **뒤**로** → 머리가 사과 위에 있는 프레임에 사과가 머리를 덮어버려서 머리가 안 보임. 사과 → 머리 순서.
- **사과를 한 번만 박고 매 프레임 안 박음** → 뱀이 지나간 자리에서 사과가 사라짐 (차분 렌더링의 `.` 처리에 같이 지워짐).
- **`#include <stdlib.h>`/`<time.h>` 누락** → `rand`/`srand`/`time` 컴파일 경고. stage-4부터 둘 다 include 해뒀으니 OK.

소리 내어 답해보기:

1. `srand`를 한 번도 안 부르면 매 실행 사과가 어디에 뜨지? 왜?
