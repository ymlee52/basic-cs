# Stage 4-(3) — 먹기 판정과 길어지는 몸

> 머리가 사과 위로 오면 **먹는다**: 길이가 늘고, 사과는 다른 자리에 새로 뜬다. 처음으로 "한 프레임만 다르게 동작" 하는 분기를 만난다.

## 오늘의 목표

- 머리가 사과 좌표에 도달하면:
  - `length++` (몸 길이 + 1)
  - 새 사과를 다른 자리에 배치
  - 그 프레임에만 **꼬리 지우기 스킵** (그래야 길어진 게 유지됨)
- 그 외 동작은 stage 4-(2)와 동일.

> **몸을 `o`로 그리는 건 다음 단계.** 이번 단계에선 길이가 늘어나도 화면엔 여전히 머리 `O` 하나만 보인다(먹은 직후 한 프레임에만 잠깐 `O`가 두 개 보일 뿐). 자료구조 안에서는 분명히 자라고 있지만, 그걸 눈에 보이게 만드는 건 stage 4-(4)에서.

기대 동작:
1. 머리가 `*` 위로 진입.
2. `*`가 사라지고 다른 자리에 새 `*`.
3. 머리는 계속 전진. 내부적으로 `length`가 1 → 2.
4. 한 번 더 먹으면 2 → 3. (눈에는 안 보임. 다음 단계에서 보이게.)

---

## 배경 이야기

### 큐의 진짜 모습 — 평소엔 한 번 넣고 한 번 빼고

Stage 4-(1)에서 큐(FIFO)를 처음 만났다. 매 프레임 우리는 사실 **enqueue(머리 추가) + dequeue(꼬리 제거)** 를 동시에 하고 있었다:

```
[꼬리 ──── 머리]   ← 이전 프레임
   dequeue ↑       ↑ enqueue
[꼬리 ──── 새머리] ← 이번 프레임
```

길이가 1이든 5든, 매 프레임 양쪽 한 개씩. → **길이가 그대로 유지**.

### 먹은 프레임에만 dequeue를 스킵하면?

```
[꼬리 ──── 머리]                    길이 N
   (스킵)         ↑ enqueue
[옛꼬리 ─── 새꼬리 ─── 새머리]      길이 N+1
```

머리는 새로 추가, 꼬리는 그대로 둠. → **길이가 1만큼 늘어남.** 이게 스네이크에서 "길어진다"의 정확한 의미.

즉, **"길어진다"는 새로 그리는 일이 아니라, 빼던 걸 한 프레임 안 빼는 일**이다. 평소 동작(enqueue+dequeue) 위에 조건부로 dequeue 한 번을 스킵.

### "한 프레임짜리 상태" 패턴

먹기는 즉시 길이를 늘리지만, 그 효과(꼬리 스킵)는 **다음 프레임 시작 시점**의 차분 렌더링에 작용해야 한다. 그래서 "이번 프레임에 먹었음" 사실을 한 변수에 잠깐 보관했다가, 다음 프레임 시작 부분에서 읽고 즉시 지운다.

```c
int ate = 0;             // 한 프레임짜리 플래그

while (running) {
  if (!ate) {            // ← 다음 프레임 시작점에서 읽음
    board[prev_tail.r][prev_tail.c] = '.';   // 꼬리 지우기
  }
  ate = 0;               // ← 읽자마자 리셋 (한 프레임만 유효)

  /* ... 렌더링, 입력, 이동 ... */

  if (먹었으면) {
    length++;
    ate = 1;             // ← 다음 프레임에서 쓸 신호
  }
}
```

같은 패턴: 자동차 깜빡이(한 번 작동 후 자동 꺼짐), GUI의 "이벤트 한 번 발사 후 소비(consume)", 게임의 "이번 턴에 받은 데미지" 임시 변수. **상태 머신의 1프레임 전이**가 다 이런 모양.

### "수명이 다른 상태"의 비유

게임 코드에는 수명이 다른 변수들이 섞여 있다:
- **영구 상태**: `length`, `body[]`, `apple` — 다음 프레임에도 계속.
- **1프레임 상태**: `ate`, 이번 프레임 `key` — 다음 프레임엔 새로.
- **즉시 소멸**: 루프 안 지역변수.

`ate`는 명시적으로 다음 프레임 진입 직후 리셋해서 1프레임 수명임을 보장한다. 안 그러면 "한 번 먹었더니 영원히 안 줄어드는 뱀"이 되어버림.

### 먹기 판정 자체는 간단

머리 좌표와 사과 좌표가 같으면 끝.

```c
if (body[0].r == apple.r && body[0].c == apple.c) {
  /* 먹었음 */
}
```

머리(`body[0]`)는 이동·clamp가 끝난 직후 위치를 검사해야 한다 — 그래야 "이 프레임에 막 진입한 자리"가 사과인지 확인됨.

### 새 꼬리는 어디로?

`length`를 1 늘렸으면 `body[length - 1]`을 채워야 한다(안 그러면 쓰레기값). 정답: **직전 꼬리 자리**.

```c
body[length - 1] = prev_tail;
```

직관: 몸이 한 칸 자라난다 = 머리는 앞으로, 꼬리는 한 프레임 동안 안 움직임. 그래서 다음 프레임 시작 시 새 꼬리는 직전 꼬리 그 자리.

### 새 사과 — `srand` 다시 안 부른다

새 사과 좌표는 `rand()` 두 번이면 됨. **`srand`는 다시 부르지 않는다.** 시드는 프로그램당 한 번. (시드를 매번 다시 깔면 같은 자리만 뽑힘 — stage 4-(2)에서 다룬 함정.)

이번 단계의 새 사과는 **몸/머리 위에 떨어질 수도 있음**(약간의 확률). 처리는 stage 4-(4)에서 — 재시도 루프(rejection sampling)로.

## 새 CS 개념

- **조건부 큐 연산**: 평소엔 enqueue+dequeue, 특정 조건에서만 dequeue 스킵 → 큐 크기가 조절됨.
- **1프레임 상태 / 이벤트 플래그**: 다음 프레임으로 단 한 번 신호 전달, 즉시 소비. 상태 머신의 가장 작은 단위.
- **변수의 수명**: 영구 / 1프레임 / 즉시 소멸. 같은 함수 안에 섞여 있어도 의도된 수명이 다르다는 자각.

---

## 새 C 문법

새로 등장하는 건 사실상 없음. 이미 본 도구들의 조합:

### 두 조건의 결합 — `&&`
- `if (a == b && c == d)` — 두 비교가 **모두 참**일 때.
- 단축 평가(short-circuit): 왼쪽이 거짓이면 오른쪽은 평가도 안 함. C에서 `if (p != NULL && p->x > 0)`처럼 안전장치로 자주 씀.

### `body[length - 1] = prev_tail;`
- 구조체 통째 대입. 필드(`.r`, `.c`)가 한 번에 복사됨. Stage 4-(1)의 `Cell prev_tail = body[0];`와 같은 매커니즘.

## 코드

`01-snake/snake.c`를 다음으로 덮어쓴다.

```c
#include <windows.h>
#include <stdio.h>
#include <conio.h>
#include <stdlib.h>
#include <time.h>

#define ROWS 8
#define COLS 18

typedef struct {
  int r;
  int c;
} Cell;

#define MAX_LEN (ROWS * COLS)

int main(void) {
  SetConsoleOutputCP(65001);

  srand(time(NULL));

  char board[ROWS][COLS];

  int dr = 0, dc = 1;

  for (int r = 0; r < ROWS; r++) {
    for (int c = 0; c < COLS; c++) {
      if (r == 0 || r == ROWS - 1 || c == 0 || c == COLS - 1) {
        board[r][c] = '#';
      } else {
        board[r][c] = '.';
      }
    }
  }

  Cell *body = malloc(sizeof(Cell) * MAX_LEN);
  int length = 1;

  body[0].r = ROWS / 2;
  body[0].c = COLS / 2;

  Cell prev_tail = body[0];

  Cell apple;
  apple.r = 1 + rand() % (ROWS - 2);
  apple.c = 1 + rand() % (COLS - 2);

  // (추가) 1프레임짜리 플래그 — "직전 프레임에 사과를 먹었다"는 신호.
  // 이번 프레임의 꼬리 지우기를 한 번만 스킵하기 위한 용도.
  // 매 프레임 시작에서 읽자마자 0으로 리셋해 수명을 한 프레임으로 강제.
  int ate = 0;

  int running = 1;

  printf("\033[?1049h");

  while (running) {
    printf("\033[H");

    // === 차분 렌더링 ===
    // (수정) 직전 프레임에 먹었으면 꼬리 지우기 스킵 → 몸이 한 칸 늘어난 채 유지.
    // board[prev_tail.r][prev_tail.c] = '.';
    if (!ate) {
      board[prev_tail.r][prev_tail.c] = '.';
    }
    ate = 0;   // (추가) 플래그 즉시 소비 — 다음 프레임엔 다시 0에서 시작.

    board[apple.r][apple.c] = '*';
    board[body[0].r][body[0].c] = 'O';

    for (int r = 0; r < ROWS; r++) {
      for (int c = 0; c < COLS; c++) {
        putchar(board[r][c]);
      }
      putchar('\n');
    }

    printf("이동: wasd, 종료: q\n");

    if (_kbhit()) {
      int key = _getch();
      switch (key) {
        case 'w': dr = -1, dc =  0; break;
        case 's': dr =  1, dc =  0; break;
        case 'a': dr =  0, dc = -1; break;
        case 'd': dr =  0, dc =  1; break;
        case 'q': running = 0; break;
      }
    }

    // === 이동 (큐 시프트 + 머리 전진) ===
    prev_tail = body[length - 1];

    for (int i = length - 1; i > 0; i--) {
      body[i] = body[i - 1];
    }

    body[0].r += dr;
    body[0].c += dc;

    if (body[0].r < 1)        body[0].r = 1;
    if (body[0].r > ROWS - 2) body[0].r = ROWS - 2;
    if (body[0].c < 1)        body[0].c = 1;
    if (body[0].c > COLS - 2) body[0].c = COLS - 2;

    // (추가) 먹기 판정 — 머리(이동·clamp 직후)가 사과 위인지.
    // 이동이 완전히 끝난 다음에 검사해야 "이 프레임에 도달한 자리"를 정확히 봄.
    if (body[0].r == apple.r && body[0].c == apple.c) {
      length++;
      // 새 꼬리는 직전 꼬리 자리 — 다음 프레임의 꼬리-스킵과 짝을 이룸.
      body[length - 1] = prev_tail;

      // 새 사과 자리. srand는 다시 부르지 않는다 (시드는 프로그램당 한 번).
      // 몸/머리 위에 떨어질 가능성은 다음 단계(4-(4))에서 처리.
      apple.r = 1 + rand() % (ROWS - 2);
      apple.c = 1 + rand() % (COLS - 2);

      ate = 1;   // 다음 프레임의 꼬리 지우기 한 번 스킵 신호.
    }

    Sleep(200);
  }

  free(body);

  printf("\033[?1049l");

  return 0;
}
```

## 돌려보기

```sh
gcc 01-snake/snake.c -o 01-snake/snake
./01-snake/snake
```

확인 사항:
- 머리 `O`가 사과 `*` 자리에 도달하면 사과가 사라지고 **다른 자리**에 새 `*`가 뜨는가?
- 새 사과 자리가 **매번 다른가**? (시드 효과)
- 먹은 직후 한 프레임에 `O`가 **두 개** 잠깐 보이는가? (꼬리 스킵의 시각적 증거 — 길이 2로 자라난 자리 + 새 머리)
- 그 외에는 stage 4-(2)와 동일하게 동작하는가? (벽 clamp, `q` 종료)

**길어지는 게 시각적으로 안 보여서 답답할 수 있다.** 자료구조는 분명히 자라고 있음(`length`가 1씩 오름). 이걸 눈에 보이게 만드는 게 다음 단계의 일.

길이 확인 디버그용 한 줄(선택, 확인 후 지우기):
```c
printf("이동: wasd, 종료: q   length=%d\n", length);
```

## 흔히 빠지는 함정

- **`ate` 리셋을 안 함** → 한 번 먹은 후 꼬리가 영원히 안 지워져서 머리가 지나간 자리에 `O` 흔적이 계속 남음. 매 프레임 시작에서 `ate = 0`.
- **`ate` 리셋 위치가 너무 늦음** → 같은 프레임 내에서 먹기 판정 직후에 리셋하면 다음 프레임의 차분 렌더링이 못 본다. **읽는 곳에서 리셋**해야 함.
- **`length++`만 하고 `body[length-1]`을 안 채움** → 새 꼬리 칸이 쓰레기값. 다음 프레임의 `prev_tail = body[length-1]`이 쓰레기를 가져옴 → 화면 어딘가에 정체불명 좌표가 `.`으로 찍히고 디버깅이 어렵다.
- **`srand`를 새 사과 뽑을 때 다시 호출** → 매번 같은 자리. stage 4-(2)에서 본 함정 재발.
- **먹기 판정을 clamp **전**에 함** → 벽을 향해 들어간 첫 프레임에 좌표가 벽 밖을 가리키는 순간에 검사 → 사과는 벽 안인데 못 만남. clamp 후에 비교해야 자연스러움.
- **시각적 확인 부재로 "안 됨" 오인** → 이번 단계는 자료구조만 자란다. `length` 출력으로 확인하지 않으면 동작하는데도 동작 안 한 줄 안다.
